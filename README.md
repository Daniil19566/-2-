1. Создать новый консольный проект
2. Определить семейство классов-продуктов:
Например, «Транспортные средства» (легковой автомобиль, грузовик, мотоцикл) или «Документы» (PDF, DOCX, XLSX) — любой набор, подходящий под конкретную задачу.
Каждый «продукт» должен реализовывать общий интерфейс или абстрактный класс, в котором будет хотя бы пара методов или свойств, уникальных для конкретного продукта.
3. Реализовать базовый класс или интерфейс Creator:
Включить в него абстрактный (или виртуальный) метод-фабрику CreateProduct(), возвращающий интерфейс или абстрактный класс продукта.
Добавить в «Создателе» любой метод, который использует фабричный метод (например, метод SomeOperation(), вызывающий CreateProduct() и демонстрирующий работу с созданным объектом).
4. Создать несколько конкретных классов (Concrete Creator):
Каждый из них переопределяет фабричный метод CreateProduct(), возвращая разные конкретные продукты из пункта 2.
Например, CarCreator, TruckCreator, MotorcycleCreator или PdfCreator, DocxCreator, XlsxCreator и т.д.
5. Подготовить отчет:
Титульный лист;
Ссылка на проект: должен содержать короткое текстовое описание (комментариями в коде или отдельным README).
Скриншоты демонстрации и комментарии.
Критерии оценки
Наличие абстрактного или виртуального фабричного метода.
Гибкое добавление новых типов продуктов без изменения существующего кода.
Корректная демонстрация (лог/консоль) работы программы: каждый «Создатель» генерирует свой объект-продукт и взаимодействует с ним.
